# The Satsuma Diaries
## Tuesday, 22 July 2026

> **tl;dr** — After a quiet few weeks, the contributors spent a day doing a proper security spring-clean: they cleared out a big pile of flagged dependencies (the third-party code the project leans on), fixed a security patch that had accidentally broken half the buttons, and got the warning board back to green.

Right, so, it's been a minute.

Last time I wrote anything it was the 0.10.0 release and everyone was tired and pleased with themselves. Then — nothing. Well, not nothing, but close. A quiet middle-of-June day where the language server (the bit that powers the red squiggles and the go-to-definition stuff in your editor) got a few fixes — it learned to properly check the messages it sends back, and to stop losing track of definitions that live in namespaces, which are basically folders for your mappings. And then the project went on what I can only describe as a six-week lie-down. Fair enough. It was hot. I wasn't here either.

Then mid-July the **dependency butler** turned up mob-handed. Quick explainer for anyone normal: every project like this stands on a tower of other people's code — little libraries that do one job each — and a bot called Dependabot watches for new versions and opens a request every time one lands. Over the 13th and 14th it opened *dozens*. Most of it is exactly as boring as it sounds — this test tool up a decimal point, that type definition up a notch — but two things in there were actually worth clocking. First, the contributors pinned every one of their automated build steps to an exact version fingerprint (a long unique code called a SHA) instead of a friendly label like "v7". Sounds pedantic. It's not: a friendly label can be quietly repointed at different code later without you noticing, and a fingerprint can't. That's a real door they shut. Second, they locked down the little preview windows the extension shows you — the "webviews" — so those windows only listen to messages from where they're actually supposed to come from, and they closed a hole in the visualiser's test rig that would cheerfully hand out any file on disk you asked it for. Good. Slightly alarming that it was ever handing out any file you asked for, but, good.

And *that's* what set up today. Because here's the thing about locking a door: sometimes you lock it on your own foot. That webview lockdown from mid-July? It broke all the webviews. Every preview panel in the extension, dead, because the new bouncer was so strict it wasn't letting the extension talk to its own windows. So the first job this morning was #386 — undo the self-inflicted bit, keep the security bit, get the panels working again. Classic. You harden the thing and the thing stops working and then you spend a morning explaining to the hardening that *we're* on the guest list.

Then the main event, which was a proper clear-out. There was a whole wall of security warnings sitting on the project — sixteen version requests backed up, most of them about the same two culprits. One's a tiny library called **brace-expansion** (it's the thing that turns `file{1,2,3}` into three filenames — genuinely that small a job) which had a bug where a nasty enough input could make it grind for ages and choke the process. A denial-of-service, in the jargon: not someone stealing your data, just someone able to make the thing seize up. The fixed version needed dropping into every single corner of the project — eight different spots — and the same story for a YAML reader (YAML being the human-friendly config format) with its own version of the same seize-up problem. Nothing dramatic to fix each one; the aggravation was that they were all wedged behind one failing safety check and refused to go in until every branch got dragged up to date with the latest code first. So that's what happened: rebase the lot, watch the checks go green in a row, merge, repeat. Seventeen requests cleared by the end of it, including one last straggler — a YAML seize-up hiding in the documentation website's build tools that nobody's bot had bothered to flag a fix for, so the contributors just did it by hand.

Net result: the warning board went from a screen of red to basically clear. No new features, nothing to click, nobody's going to notice — which is exactly the point of a day like this. It's washing-up. Someone's got to.

I go away for six weeks and come back to a spring-clean. On brand, honestly.
