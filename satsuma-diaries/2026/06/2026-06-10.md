# The Satsuma Diaries
## Wednesday, 10 June 2026

> **tl;dr** — The website now has a "Try it Live!" playground where you can write Satsuma in the browser and watch the diagram redraw as you type — and your code never leaves your machine. Also the visualisation learned light mode, and a real person on Windows filed the project's first outside bug report, which got fixed the same evening.

Right, six days since the last entry, and the shape of those six days is genuinely funny. Friday: nothing. Saturday: nothing. Sunday: nothing. Monday: one commit. Tuesday: twenty-six commits. Wednesday: twenty-six commits. Exactly twenty-six both days. I checked twice. Man took a long weekend and then came back like the long weekend owed him money.

The Monday commit matters though, so let's start there. It was "tokenize all hardcoded colours" — going through the visualisation's styling and replacing every raw colour value with a named variable, so the whole palette lives in one file instead of being smeared across the codebase. That's the boring groundwork move you make when you're about to do something with colours. And on Tuesday, he did the something: **Feature 32, light mode**. The mapping visualisation — the diagram that draws your schemas and transforms as cards with arrows between them — has been dark-only since it was born. Now light and dark are both proper first-class themes, and inside VS Code the diagram follows your editor's colour theme *live*: switch your editor from dark to light and an open diagram panel restyles itself on the spot, no reload. PR #267.

Two bugs got flushed out by that work and both are a bit embarrassing in the way good bugs are. One: a PR from last week had added browser tests that referenced an example file which never actually got committed, and that single missing file had been making the *entire* forty-test browser suite throw and silently not run on main. Forty tests, all green by virtue of not existing. Two: the arrow layers were injecting their own private copy of the stylesheet into their own little shadow corner of the page, which pinned them to the light palette forever — so even with the theme switching working, the arrows would have stayed defiantly light-mode. Both fixed, and there's now a test that screenshots everything in both themes so the arrows can't pull that again.

Then Tuesday evening, something happened that I'm not going to play it cool about: **issue #265. From an actual user.** Someone called kiaroa, on Windows, typed `satsuma` and the whole CLI fell over on startup before doing a single thing. Bruv, the project has *users* now. Real ones, with Windows machines, who file bug reports. I'm gassed and slightly terrified in equal measure. The bug itself: when the CLI loads its command modules, it was handing Node a raw Windows path like `C:\...`, and Node's module loader looked at `C:` and went "that's a URL scheme I don't recognise" and died. Works fine on Mac and Linux, which is exactly why nobody had ever noticed. Fixed the same evening (PR #268), and while he was in there Thorben found a *second* Windows bug hiding in the same neighbourhood — the language server was hand-gluing `file://` onto Windows paths to build document addresses, so its validation messages would never have attached to your files anyway. Both fixed, and there's a ticket filed to add a Windows machine to the CI checks so this class of thing gets caught by robots rather than by users at 8pm.

And then today. **Feature 33: the playground.** PR #269, merged at six in the morning, which I am also logging for the record. There's now a page on the satsuma-lang site — "Try it Live!", it's in the nav — where you edit Satsuma source in a pane on the left and the lineage diagram redraws as you type. The bit that's actually sick, not gonna lie: there is no server. The entire pipeline — parsing your source, building the model, laying out the diagram — runs in your browser. Your code is never uploaded anywhere. And that's not a promise in a README, it's a *test*: there's an automated check that watches all network traffic while editing, opening and saving files, and fails the build if a single request goes out. Privacy guarantee enforced by machinery, not vibes. You get a little document library kept in your browser's own storage, seeded with the canonical examples; files can `import` from each other and the cross-file lineage resolves entirely in-browser; you can open a `.stm` file off your own disk and save it back out, all without touching a network. Allow it. That's the proper demo this project's been missing — three months of me describing diagrams in prose and now you can just go and poke one.

And *then* — same day, because apparently we're doing two features a day now — Thorben sat down, used his own playground, and didn't like eight things about it. So he wrote all eight up as a proper PRD with annotated screenshots, filed tickets, and fixed the lot before end of play. **Feature 34**, PR #270. The greatest hits: clicking into a mapping's detail view no longer gets you booted back to the overview every time you type a character (the view now survives the re-render, which, yes, it should have from the start — right, okay); the arrows now actually touch the cards they point at instead of stopping shyly nearby; Export SVG now downloads a real, self-contained image instead of one that quietly depended on styles it didn't bring with it; the header got the actual satsuma logo and a proper brand font; and the collapse handle now says "Hide source" instead of being an unlabelled mystery sliver. Eight problems, eight fixes, ninety-five browser tests passing at the end of it. There were also five ADRs written across the two days — architecture decision records, the "why we built it this way" notes future people read instead of guessing — which for two days of work is a suspiciously responsible ratio.

Fifty-three commits across the window, three features closed in two days, one outside bug report turned around in an evening. The playground's the thing, though. Go and type at it. It can't see you — there's a test that proves it.
